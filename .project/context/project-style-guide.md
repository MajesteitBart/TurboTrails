# Project Style Guide

## Visual Direction

Cartoon, bright, child-friendly, readable, and original. Use 2.5D parallax depth where useful. Interactive objects must stand out more than decoration.

## UI Direction

Gameplay uses a dense HUD: progress bar, timer, stunt message, chest counter, pause, touch controls, combo, and optional debug values. Touch controls must be large enough for children and support multi-touch.

## Code Direction

- Prefer explicit TypeScript interfaces for data contracts.
- Keep comments sparse and focused on physics, stunt, or save behavior that is not obvious.
- Keep levels and balancing values data-driven.
- Avoid backend assumptions.
